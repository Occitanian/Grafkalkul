## 📌 О проекте

Данный проект представляет собой графический калькулятор, который отображает трёхмерные графики функций двух переменных _z = f(x, y)_.  
В отличие от традиционного построения полигональной сетки, здесь используется **ray marching** – итеративный метод поиска пересечения луча с неявно заданной поверхностью. Это позволяет получать гладкие, детализированные изображения сложных математических форм.

Проект реализован в **Construct 3** с активным использованием чистого **JavaScript** для алгоритмической части. Большая часть логики ray marching вынесена в отдельный модуль, что делает код прозрачным и легко расширяемым.

## ✨ Возможности

- Построение 3D-графиков по введённой пользователем формуле (поддерживаются операторы `+ - * / ^`, функции `sin`, `cos`, `tan`, `abs`, `log10`, константа π).
- Предустановленные поверхности:
  - **Сфера** (по кнопке `SPH`)
  - **Конус / эллиптический параболоид** (`EPA`)
  - **Гиперболический параболоид** (`HPA`)
  - **Волны синуса и косинуса** (`WAW`)
  - **Корневая волна** (`ROF`)
  - **«Корона»** (`CRO`)
  - **Квадратные волны** (`ISH`)
  - **Просто число 67** (`SS`) – демонстрация плоской поверхности.
- Свободное вращение камеры с помощью мыши (зажатая левая кнопка + движение).
- Приближение / отдаление колесом мыши.
- Полноэкранный интерфейс с кнопочной панелью для ввода и выбора функций.

## 🧠 Алгоритмическая основа: Ray Marching

### 🔍 Что такое ray marching?

Ray marching – это метод рендеринга, при котором для каждого пикселя экрана пускается луч, и сцена «маршируется» (шаг за шагом) до тех пор, пока не будет найдено пересечение с поверхностью. В отличие от классической трассировки лучей, здесь не требуется аналитического решения пересечения – достаточно иметь **функцию расстояния со знаком (SDF)** для каждого объекта.

### 📐 Реализация в проекте

В файле [`scriptsInEvents.js`](scripts/scriptsInEvents.js) содержится полный модуль ray marching, включающий:

#### 1. **SDF-примитивы**
   - `sdSphere(p, r)` – сфера
   - `sdBox(p, b)` – параллелепипед
   - `sdTorus(p, r1, r2)` – тор

#### 2. **Комбинирование сцены**
   Функция `mapWorld(p)` возвращает минимальное расстояние до ближайшего объекта (объединение нескольких примитивов).

#### 3. **Вычисление нормали**
   `calcNormal(p)` – градиент SDF в точке (используется для освещения).

#### 4. **Освещение (Phong)**
   `phongLighting(p, viewDir, lightPos)` – расчёт цвета с учётом диффузной и зеркальной составляющих.

#### 5. **Основной цикл ray marching**
   `rayMarch(origin, direction)` – итеративно двигается вдоль луча, пока не достигнет поверхности или не превысит максимальное расстояние.

#### 6. **Генерация лучей камеры**
   `getCameraRay(x, y, width, height, camPos, camTarget, camUp, fov)` – строит луч для каждого пикселя в соответствии с параметрами виртуальной камеры.

#### 7. **Функция полного рендера**
   `renderRayMarchedScene(width, height)` – создаёт canvas с изображением сцены (для демонстрационных целей).

> ⚠️ **Примечание**: В текущей версии графики основной рендеринг выполняется через встроенные средства Construct 3 и поведение `sphere`, но приведённый код полностью описывает альтернативный ray marching-подход и служит подтверждением понимания алгоритма.

## 🕹️ Управление

| Действие | Результат |
|----------|-----------|
| Зажать левую кнопку мыши и двигать | Вращение сцены по горизонтали / вертикали |

## 🛠️ Технические детали

- **Платформа**: [Construct 3](https://www.construct.net) (экспорт в HTML5)
- **Язык программирования**: JavaScript (основные алгоритмы), события Construct 3 (интерфейс)
- **Архитектура**:
  - Объект `Graph` – основная 3D-поверхность, на которой строится график.
  - Поведение `Sphere` – задаёт mesh сферы (используется как основа для деформации).
  - Скриптовые функции в `scriptsInEvents.js` – обработка математических выражений и ray marching.
- **Математические выражения** обрабатываются через `eval()` с предварительной заменой токенов (`X` → числовое значение, `SIN` → `Math.sin` и т.д.). Для безопасности ввод ограничен экранной клавиатурой.
